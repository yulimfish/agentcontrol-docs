# 安全边界

AgentControl 的设计原则是：**只控制用户自己的本地客户端，不触碰服务器权限**。

## 不会做的事情

- ❌ 绕过服务器权限或 OP 判定
- ❌ 绕过反作弊系统
- ❌ 绕过区域保护或领地插件
- ❌ 绕过游戏规则或冷却时间
- ❌ 以机器人账号加入服务器
- ❌ 安装服务端插件或修改服务器

## 会做的事情

- ✅ 读取本地客户端可见的状态（坐标、血量、物品栏等）
- ✅ 通过正常客户端 API 发送按键、点击、视角调整
- ✅ 在本地 `127.0.0.1` 暴露端点供 MCP 调用
- ✅ 配合 Mod Menu 提供本地配置选项

## 调用链路

```text
AI 助手
  -> agentcontrol-mcp (本地 MCP 服务)
  -> http://127.0.0.1:17777
  -> agentcontrol-fabric (本地 Fabric 模组)
  -> Minecraft 客户端 API
  -> 远程服务器 (正常网络包)
```

服务器接收到的仍然是**正常的客户端操作包**，和真人玩家没有区别。

## 建议

- 在公共服务器使用时，保持动作有界、频率合理
- 不要用于自动化刷资源或挂机，可能违反服务器规则
- 单人世界或私人服务器可自由使用
